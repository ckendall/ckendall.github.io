Starting DevBootcamp

12-3-15

Hello all! We are back after a couple days or weeks, they are all starting to blend together on the learning adventure that I have embarked upon. Oh well anyways, what I have been learning this w ek is how to make classes and how to work with classes. I have been learning a lot and ever since I started learning Ruby, classes h ve always seemed a little hard to grasp, but now that I am working ith them a lot more they are getting easier for me to understand. Lets get straight into an example shall we?

Here is and example of an animal class for any type of animal

class Animal

attr_reader :type, :age, :color

def initialize(type, age, color)

@type = type

@age = age

@color = color

end

def hello

puts “hello I’m a #{@type} and my color is #{@color} and I am #{@age} years old”

end

def grow(years)

@age += years

end

def change(fur_color)

if @age > 7

@color = fur_color

end

end

end

This is an example of an animal class that will take in age color and type of animal it is. If I wanted to make a new instance of this class I would call it like this

dog = Animal.new(chow, 4, black)

this would set my variables up in the initialize method to their respective spaces. If I wanted to call on hello I would do this: dog.hello and it would print out the sentence in the quotes with the variables for the dog instance. Now all of the variables so far are called instance variables. They are instances of the class Animal and I could make new ones by creating a new class like this.

cat = Animal.new(cat, 8, black)

I just made a new animal and for this instance my type, age and color variables would change to these. Instance variables are really nice because we can assign them new values, but also keep them separate from others and they are denoted with the @ symbol. This is so we or the computer don’t get screwed up on what we are doing. Instance variables are also able to be seen within our class animal, as you can see I have my @age in a couple different spots in the class. I need these to be flexible and move where I need them. Any variable that I want to be flexible in a class I need to make it an instance variable. Unlike the local variable years and fur_color. If I tried putting these anywhere else in the program I would get an error telling me no, and that’s no fun, so I need to abide by the laws of the program.

There is a lot more to classes and everything about them that I am still learning, but this is the basics that I have learned and I am glad to show them to you!! I hope everything is going well for all of you and strive to learn something new everyday! Talk to you again soon, Cody.
